# Project Goals

## Primary Goals

1. **Design a modular cluster architecture** that can be assembled from heterogeneous, commercially available hardware components
2. **Implement a software stack** for cluster management, job scheduling, and resource allocation
3. **Provide user-friendly interfaces** for researchers to submit, monitor, and manage AI/ML workloads
4. **Evaluate cluster performance** on representative machine learning tasks and benchmarks

## Secondary Goals

- Ensure reproducibility of experiments through containerization and infrastructure-as-code
- Minimize operational costs while maximizing utilization of available hardware
- Document the deployment process so other institutions can replicate the setup
- Integrate monitoring and observability tools for cluster health and usage analytics

## Success Criteria

- The cluster can run distributed training jobs across multiple nodes
- Resource utilization is significantly improved compared to standalone workstations
- The system is usable by researchers without deep DevOps knowledge
- Total cost remains within a budget feasible for a university department
